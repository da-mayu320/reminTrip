## サービス概要
複数サイトで予約した旅行の予約情報（時間、場所、予約番号）を一元管理し、
メールで探す手間なく、いつでもすぐに確認できるアプリです。

## このサービスへの思い・作りたい理由
ヨーロッパ周遊一人旅を2週間行った際に、いろんなサイトから、移動手段、ホテルを予約しており、何度も予約確認メールを検索したり、自分が探している情報を見つけるのが手間だった。
心配性であることから、何度も予約があっているか、時間や乗り場などの確認が大変であったという経験があった。
これらの経験から手間や心配を少しでもなくし、旅行を充実したものにしたいという思いからこのサービスを作りたいと考えた。

## ユーザー層について
複数サイトで予約を取るため、情報が散らばって管理が大変な人 
→いろんな媒体から予約していることが想像されるから

## サービスの利用イメージ
移動手段などの情報を登録し、時間、乗り場、場所など重要度の高い情報をメールを開くことなくアプリで一括で見ることができることにより、情報取得の手間をなくすという価値を得ることができる。

## ユーザーの獲得について
・旅行系インフルエンサー（X,インスタグラム、ブログ、YouTuberなど）や、旅行系のコニュニティーへの発信（無償で使ってもらってレビューを依頼）
・実際の予定登録画面、リマインドの様子など実際の使用様子を発信してもらう。

## サービスの差別化ポイント・推しポイント
Gmailなどからカレンダーに予定登録なども可能ではあるが、
日時のみの登録しかできなかったり、正確な時間が登録できない、
また、メールの様式によっては直接カレンダーに追加できないこともあり
指定の情報をメールから自動で読み取り、一覧で見られるようにすることで、旅行に必要な情報が一度で閲覧できるという点が差別化ポイントである。

## 機能候補
【MVPリリース(2週間以内)】
・メールから情報を読み取る（時間、場所、予約番号）
・メールから取得した情報を一覧表示（時系列で一覧表示（日時が近い順））
・会員登録・ログイン機能の追加
【本リリース(卒業認定後)】
- QRコード表示機能
- LINE通知機能

## 機能の実装方針
- 予定の一覧表示: ActiveRecordのorderメソッドで時系列に並び替え
