# README

## サービス概要
旅行の予約情報が複数のメールに散らばっていて確認が大変な人に
重要な予定を一括管理し、希望のタイミングでリマインドする価値を与える
旅行予定管理Webアプリです

## このサービスへの思い・作りたい理由
ヨーロッパ周遊一人旅を2週間行った際に、いろんなサイトから、移動手段、ホテルを予約しており、
何度もメールを検索して手間だった。
心配性であることから、何度も予約があっているか、時間や乗り場などの確認が大変であったという経験があった。
これらの経験から手間や心配を少しでもなくし、旅行を充実したものにしたいという思いからこのサービスを作りたいと考えた。

## ユーザー層について
複数サイトで予約を取るため、情報が散らばって管理が大変な人 
→いろんな媒体から予約していることが想像されるから

## サービスの利用イメージ
移動手段などの情報を登録し、リマインダー機能により現在の運行状況や乗り場、場所など重要度の高い情報をメールを開くことなくアプリで一括で取得できることにより、情報取得の手間をなくすという価値を得ることができる。

## ユーザーの獲得について
・旅行系インフルエンサー（X,インスタグラム、ブログ、YouTuberなど）や、旅行系のコニュニティーへの発信（無償で使ってもらってレビューを依頼）
・実際の予定登録画面、リマインドの様子など実際の使用様子を発信してもらう。
## サービスの差別化ポイント・推しポイント
人によって、リマインドを欲しいタイミングはそれぞれであるため、どの内容をいつ通知するのかは自分で設定できるようにする点。
（ホテルの場所やチェックインの時間は空港に着いたときでよいなど）
→出発の2時間前に知りたい人、前日に知りたい人、その人の求めているタイミングで知りたい情報を自分で選択できるところが優れている。

## 機能候補
【MVPリリース(2週間以内)】
・予定の登録・編集・削除
・予定の一覧表示（時系列で一覧表示（日時が近い順））
・予定の種類ごとの分類(移動・ホテル・レストランなど)
・予定の詳細表示(時間・場所・予約番号など)
・会員登録・ログイン
【本リリース(卒業認定後)】
予定ごとに個別にリマインドタイミングを設定
リマインド通知の配信
Googleマップ連携

## 機能の実装方針
- 会員登録・ログイン機能: Sorceryを使用
- 予定の一覧表示: ActiveRecordのorderメソッドで時系列に並び替え
- 予定の種類ごとの分類: プルダウンで選択可能にする

## 使用する技術スタック
- 使用するフレームワーク: Rails 7.0.3
- 言語: Ruby 3.1.4
- データベース: PostgreSQL
- デプロイ先: Render
- 使用予定のライブラリ: Sorcery(会員登録・ログイン機能で使用)
